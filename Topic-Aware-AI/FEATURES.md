# Feature: LDA Topic Modelling (Gensim)

## Overview
Builds an interpretable **topic model** over 2,000 IMDB reviews using Latent Dirichlet Allocation.

## Implementation Details

### Files
- `step2_preprocess.py` — tokenisation + stop-word removal
- `step3_lda_topics.py` — dictionary, corpus, LDA training

### Config
| Parameter | Value | Notes |
|-----------|-------|-------|
| `num_topics` | 5 | Discoverable thematic buckets |
| `passes` | 10 | Training epochs over corpus |
| `alpha` | auto | Asymmetric Dirichlet prior on docs |
| `eta` | auto | Asymmetric Dirichlet prior on words |
| `no_below` | 5 | Prune tokens seen in < 5 docs |
| `no_above` | 0.7 | Prune tokens seen in > 70% of docs |

### Output
```
data/topic_distributions.json   # [[0.02, 0.70, ...], ...]   2000 × 5
models/lda_model.gensim
models/lda_dictionary.gensim
```

### Coherence Score (c_v)
Measures the semantic quality of learned topics (higher = better).  
Expected range: **0.40 – 0.65** for 5 topics on IMDB.

# Feature: DistilBERT Semantic Embeddings

## Overview
Dense contextual embeddings generated by mean-pooling the **DistilBERT** last hidden state.

## Implementation Details

### File
- `step4_bert_embeddings.py`

### Architecture Decision: Mean Pooling
```
tokens           → DistilBERT → last_hidden_state [batch, seq_len, 768]
attention_mask   → masked mean → sentence vector  [batch, 768]
```
Mean pooling is more stable than `[CLS]` pooling for semantic similarity tasks.

### Config
| Parameter | Value |
|-----------|-------|
| Model | `distilbert-base-uncased` |
| Max length | 256 tokens |
| Batch size | 16 |
| Device | Auto (CUDA / CPU) |
| Embedding dim | 768 |

### Output
```
data/bert_embeddings.npy    # float32, shape (2000, 768)
```

# Feature: Hybrid Feature Fusion

## Overview
Fuses LDA and BERT into a single **773-dimensional hybrid vector** per document.

## Implementation Details

### File
- `step5_combine_features.py`

### Fusion Formula
```python
hybrid(doc) = L2_norm(BERT_768) ⊕ L2_norm(LDA_5)
            =  shape [773]
```
Both components are L2-normalised before concatenation so neither dominates by scale.

### Why L2 Normalisation?
BERT vectors have magnitudes ≈ 10–30; LDA vectors sum to 1.  
Without normalisation, BERT would dominate the cosine similarity calculation.

### Output
```
data/hybrid_features.npy   # float32, shape (2000, 773)
data/labels.npy            # int,     shape (2000,)
```

# Feature: Semantic Search Engine

## Overview
Top-k cosine similarity search over the corpus using hybrid vectors.

## Implementation Details

### File
- `step6_similarity_search.py`

### How It Works
1. Embed query with DistilBERT (same batched code as step 4)
2. Get LDA topic distribution for query
3. Fuse → 773-dim hybrid query vector
4. `cosine_similarity(query_vec, corpus_matrix)` → ranked results

### CLI Usage
```bash
python step6_similarity_search.py --query "horror film with twist ending" --top_k 5
```

### Expected Output
```
[1] Similarity: 0.9123  |  Positive ✅
     "This was one of the most terrifying movies I've ever watched..."
```
